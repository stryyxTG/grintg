# TGLol Bot

Telegram-бот на `aiogram` + `Telethon` для одиночного хранения Telegram-аккаунтов и работы с файлами `.session` / `.json`.

Используй бот только для аккаунтов, к которым у тебя есть законный доступ.

## Возможности

- Добавление аккаунтов по номеру телефона и коду входа.
- Массовое добавление аккаунтов через `.zip`.
- Умное сопоставление `.session` и `.json` внутри ZIP.
- Сохранение оригинального JSON без обрезания полей.
- Автогенерация desktop JSON, если в ZIP для аккаунта не было JSON.
- Единое хранилище аккаунтов с разделами `НЕРЕГ` и `РЕГ`.
- Метки сервисов для зарегистрированных аккаунтов: `IMO`, `BEBE`, `WHOOSH`. Один аккаунт можно отметить сразу в нескольких сервисах.
- Отдельное меню фильтра раздела `РЕГ` по сервису и по исключению сервиса.
- Скачивание аккаунтов из хранилища ZIP-архивом.
- Удаление аккаунтов по одному или массово по разделу.
- Поддержка нескольких владельцев через `OWNER_IDS`.

## Требования

- Python 3.11+
- Telegram Bot Token от `@BotFather`
- `TELEGRAM_API_ID` и `TELEGRAM_API_HASH` с https://my.telegram.org

## Установка

```bash
git clone https://github.com/stryyxTG/tglol.git
cd tglol

python3 -m venv venv
source venv/bin/activate

pip install -r requirements.txt
```

Для Windows:

```powershell
python -m venv venv
.\venv\Scripts\activate
pip install -r requirements.txt
```

## Настройка `.env`

Создай файл `.env` в корне проекта:

```env
BOT_TOKEN=PUT_BOT_TOKEN_HERE
OWNER_IDS=123456789,987654321

TELEGRAM_API_ID=123456
TELEGRAM_API_HASH=put_api_hash_here

BOT_PARSE_MODE=HTML
DATA_DIR=storage
```

Можно использовать `ADMIN_IDS`, но лучше указывать владельцев через `OWNER_IDS`.

Несколько владельцев указываются через запятую, пробел или точку с запятой:

```env
OWNER_IDS=111111111,222222222,333333333
```

## Запуск

```bash
python bot.py
```

Запуск на сервере через `nohup`:

```bash
source venv/bin/activate
nohup python bot.py > bot.log 2>&1 &
```

Проверить лог:

```bash
tail -f bot.log
```

Остановить процесс:

```bash
ps aux | grep bot.py
kill PROCESS_ID
```

## Как пользоваться

1. Напиши боту `/start` с аккаунта владельца.
2. Открой раздел `Аккаунты`.
3. Добавь аккаунт одним из доступных способов.
4. Все новые аккаунты сначала попадают в `НЕРЕГ`.
5. В карточке аккаунта можно перенести его в `РЕГ`, выбрав один или несколько сервисов, или обратно в `НЕРЕГ`.

## Добавление аккаунтов

Доступные способы:

- `По номеру + код`
- `Массово .zip`

При добавлении через ZIP архив может быть плоским, без папок. JSON может быть рядом с `.session`, а может отсутствовать:

```text
79586161128.session
79586161128.json
79586163636.session
```

Если JSON есть, бот сохраняет его оригинальным и при скачивании отдаёт именно его.
Если JSON для конкретного `.session` нет, бот генерирует desktop JSON на Windows/Linux профиле.

## Хранилище

В боте есть два раздела:

- `НЕРЕГ` — новые или ещё не отмеченные как зарегистрированные аккаунты.
- `РЕГ` — аккаунты, которые владелец вручную отметил как зарегистрированные.

При переносе аккаунта в `РЕГ` нужно выбрать один или несколько сервисов:

- `IMO`
- `BEBE`
- `WHOOSH`

В списке аккаунтов раздела `РЕГ` выбранные сервисы показываются прямо в кнопке аккаунта. Внутри `РЕГ` есть отдельная кнопка фильтра.

Фильтр поддерживает:

- `Все РЕГ`
- `IMO`
- `не IMO`
- `BEBE`
- `не BEBE`
- `WHOOSH`
- `не WHOOSH`

Например, `IMO` покажет аккаунты, где среди меток есть IMO, а `не IMO` покажет зарегистрированные аккаунты без метки IMO и старые `РЕГ` без указанного сервиса.

Для каждого аккаунта можно:

- открыть карточку;
- проверить, живая ли текущая `.session`, с отдельным подтверждением;
- скопировать номер;
- получить последний код Telegram из сервисного чата аккаунта;
- получить `Verification Code` из `Verification Codes`;
- скачать `.session`;
- скачать JSON;
- перенести между `НЕРЕГ` и `РЕГ`;
- сменить список сервисов, если аккаунт уже в `РЕГ`;
- удалить аккаунт и его файлы с сервера.

Для всего раздела можно скачать ZIP или удалить все аккаунты раздела.

## Хранилище файлов

По умолчанию данные лежат в папке `storage`:

```text
storage/
  bot.sqlite3
  sessions/
  json/
  tmp/
```

Что где лежит:

- `bot.sqlite3` — база данных.
- `sessions/` — `.session` файлы аккаунтов.
- `json/` — оригинальные или сгенерированные JSON.
- `tmp/` — временные файлы загрузки.

При удалении аккаунта из бота строка удаляется из базы, а файлы аккаунта удаляются с сервера. Telegram-сессия в самом Telegram при этом не разлогинивается.

## Миграция со старой версии

Если в старой базе были аккаунты, выданные другим разделам, при запуске новой версии бот вернёт их в единое хранилище. Старые служебные таблицы больше не используются.
